Retell webhook payload structure (tool_call format from Retell docs):
```json
{
  "event": "tool_call",
  "tool_call_id": "call_abc123",
  "name": "answer_question",
  "arguments": {
    "client_id": "acme-appliance",
    "question": "What are your hours on Saturday?"
  }
}
```

Retell expects tool call response:
```json
{
  "tool_call_id": "call_abc123",
  "result": "We're open Saturday 9am to 2pm."
}
```

For transfer: Retell's transfer_call tool uses a different call path. For the fallback from answer_question, the RAG service returns a sentinel string; the webhook handler then returns a transfer_call response to Retell:
```json
{
  "tool_call_id": "call_abc123",
  "result": "__TRANSFER__"
}
```
The Retell agent's LLM prompt (configured outside this codebase) interprets "__TRANSFER__" as a cue to invoke its transfer_call tool. This is the lightest integration path — no direct Retell transfer API call needed from the webhook.

**Create voice-agent/app/services/rag.py:**

Constants:

**async retrieve_chunks(db: AsyncSession, client_id: str, question_embedding: list[float]) -> list[str]:**
- Query embeddings table using pgvector cosine distance operator <=>
- SQL: SELECT content FROM embeddings WHERE client_id = :client_id ORDER BY embedding <=> :q_vec LIMIT :top_n
- Use sqlalchemy text() for this raw SQL (pgvector operator not natively in SQLAlchemy ORM)
- Only return chunks where cosine similarity >= SIMILARITY_THRESHOLD
  - Cosine similarity = 1 - cosine_distance
  - Filter: WHERE (1 - (embedding <=> :q_vec)) >= :threshold
- Return list of content strings (may be empty if no chunks pass threshold)

**async generate_answer(question: str, chunks: list[str]) -> str:**
- Use anthropic.AsyncAnthropic(api_key=get_settings().anthropic_api_key)
- System prompt (keep it natural for phone conversation):
  ```
  You are a helpful receptionist at an appliance repair shop. Answer the caller's question
  naturally and conversationally, as a knowledgeable staff member would on the phone.
  Base your answer only on the provided context. Keep answers concise but complete.
  Never mention documents, sources, or that you looked anything up.
  If the context doesn't contain enough information to answer confidently, say only: __TRANSFER__
  ```
- User message: "Context:\n{chunks_joined}\n\nQuestion: {question}"
  - chunks_joined = "\n\n".join(chunks)
- Call client.messages.create(model=HAIKU_MODEL, max_tokens=300, messages=[...])
- Return response.content[0].text.strip()

**async answer_question(db: AsyncSession, client_id: str, question: str) -> str:**
- Embed the question using openai.AsyncOpenAI(api_key=get_settings().openai_api_key)
  - Call client.embeddings.create(model="text-embedding-3-small", input=[question])
  - Extract: q_embedding = response.data[0].embedding
- Call retrieve_chunks(db, client_id, q_embedding)
- If len(chunks) == 0: return TRANSFER_SENTINEL  # No content above threshold — transfer
- Call generate_answer(question, chunks)
- If result == TRANSFER_SENTINEL or "__TRANSFER__" in result: return TRANSFER_SENTINEL
- Return the answer string

All functions must have full type annotations. Use logging (not print).

```python
@router.post("/webhook")
async def retell_webhook(request: Request, db: AsyncSession = Depends(get_db)):
    body = await request.json()
    
    # Only handle tool_call events; ignore others
    if body.get("event") != "tool_call":
        return {"status": "received"}
    
    tool_name = body.get("name")
    tool_call_id = body.get("tool_call_id", "")
    args = body.get("arguments", {})
    
    if tool_name == "answer_question":
        client_id = args.get("client_id", "")
        question = args.get("question", "")
        result = await answer_question(db, client_id, question)
        return {"tool_call_id": tool_call_id, "result": result}
    
    # Unknown tool — return not_implemented but don't crash
    return {"tool_call_id": tool_call_id, "result": "not_implemented"}
```
